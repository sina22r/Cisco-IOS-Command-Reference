# Contributing Guide

Thank you for your interest in contributing.

Our goal is to build one of the highest quality Cisco IOS references available.

---

# Before Contributing

Please ensure that:

- Information is technically accurate.
- Examples have been tested whenever possible.
- Markdown formatting is consistent.
- Grammar and spelling are correct.

---

# Documentation Style

Every command should follow this structure:

```markdown
# Command Name

## Description

## Syntax

## Abbreviation

## Privilege Level

## Configuration Mode

## Parameters

## Examples

## Example Output

## Verification

## Troubleshooting

## Common Mistakes

## Best Practices

## Security Notes

## Performance Notes

## CCNA Notes

## CCNP ENCOR Notes

## Real-World Notes

## Related Commands
```

---

# Markdown Rules

- Use ATX headings (`#`, `##`, `###`).
- Use fenced code blocks.
- Use tables where appropriate.
- Keep line length readable.
- Use English only.

---

# Code Style

Cisco CLI examples should use:

```cisco
show ip interface brief
```

Example output should use:

```text
GigabitEthernet0/0 is up
```

---

# Pull Requests

Please include:

- Clear title
- Description
- Reason for change
- References (if applicable)

---

# Issues

Please report:

- Incorrect syntax
- Missing commands
- Broken examples
- Typographical errors
- IOS version differences

---

# Code of Conduct

Be respectful.

Constructive feedback is always appreciated.
